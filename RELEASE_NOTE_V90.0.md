🚀 Mouse4 V90.0 - 终极纯净版：架构重塑与全域稳定 (The Ultimate Clean)

经过多轮版本迭代与技术验证，我们带来了 Mouse4 史上架构最精简、运行最稳定的版本 V90.0！

本次更新是一次彻底的"架构净化"——移除了所有实验性的冗余桥接层和延迟加载策略，回归最纯粹的单文件运行哲学。同时集成了此前多个版本的成熟修复成果，将稳定性推向新高度。从底层热键注册到上层 UI 绘制，每一个环节都经过了重新审视与加固。

✨ 核心亮点 (Highlights)

🛡️ 全域崩溃拦截网 (Black Box)
不再容忍静默死亡。V90.0 挂载了 `sys.excepthook` + `threading.excepthook` 双层异常拦截器。无论是主线程还是后台守护线程发生未处理崩溃，程序都会自动弹出可视化报警对话框，精准定位错误并写入日志。每一次崩溃，都有迹可循。

⚡ 原生热键 + 强力重启双重保障
采用 Windows 官方 `RegisterHotKey` API 注册全局热键 Ctrl+1，热键直接挂载于内核消息队列，免疫睡眠唤醒。同时看门狗 (Watchdog) 每 5 秒侦测时间跳变，一旦检测到深度休眠后的系统苏醒，立即执行 Hard Restart 启动全新进程，确保热键 100% 抢占成功。

🖱️ "不死图腾" 鼠标监听
为资源管理器"双击空白处返回"功能移除了脆弱的局部导入模式，将 `pynput` 提前至模块最顶层加载并锁定内存地址。无论系统休眠多久，唤醒后监听线程依然健壮如初。

📋 纯净剪贴板方案
移除了冗余的 QBuffer/QIODevice 桥接层，回归 PIL 直写方案。在内存中精准生成标准 DIB 格式并剥离 BMP 文件头，截图粘贴 100% 成功率，干净利落。

🐛 错误修复 (Bug Fixes)

- 修复 电脑睡眠唤醒后 `pynput` 报 `ImportError` 导致监听线程崩溃的问题
- 修复 截图选区靠近屏幕底部时，工具栏溢出不可见的问题（智能反弹至选区上方）
- 修复 托盘图标运行数小时后随机消失的问题（全局强引用防止 GC 回收）
- 修复 DPI 手动设置与 `uiautomation` 的时序冲突告警
- 移除 `Failed to remove temporary directory` 和 `_imaging` 加载失败的报错弹窗

🛠️ 技术细节 (Under the Hood)

**架构重塑**：彻底移除了实验性的"延迟加载 (Lazy Load)"，恢复所有重型库的全局前置导入，确保程序一旦启动运行环境就 100% 稳固。废弃了冗余的 QBuffer 内存中转层，减少不必要的对象生命周期管理。

**告别系统垃圾**：恢复最标准的单文件 (-F) 运行模式。用完即走，绝不在用户的 AppData 中留下程序解压残骸（仅保留轻量级 `config.json` 和运行日志）。

**UI 智能适配**：工具栏坐标计算逻辑重写，自动检测屏幕边界并智能选择弹出方向，从此告别截图工具栏"跑出屏幕"的尴尬。
